# ChatGPT-style 3D Icon

Parametric CadQuery model for a small printable ChatGPT-style icon: a round base with a raised six-part folded-band logo and clear central hexagon negative space.  The base diameter is computed automatically from the logo bounding box plus a configurable margin per side, so the base stays snug around the logo.

## Install dependencies

Use Miniconda/conda so CadQuery's native CAD dependencies come from conda-forge:

```bash
cd /home/miku/projects/chatgpt_icon
conda env create -f environment.yml
conda activate chatgpt-icon-cad
```

If the environment already exists and you changed dependencies:

```bash
conda env update -f environment.yml --prune
conda activate chatgpt-icon-cad
```

## Generate the model

```bash
cd /home/miku/projects/chatgpt_icon
python models/chatgpt_icon.py
```

The script creates:

- `exports/chatgpt_icon_single.stl` — combined base + logo in shared coordinates
- `exports/chatgpt_icon.step` — STEP assembly with separate `base` and `logo` solids
- `exports/chatgpt_icon_base.stl` — base only
- `exports/chatgpt_icon_logo.stl` — raised logo only, kept at the correct Z position for alignment
- `exports/preview.png` — dependency-free top-view preview generated by the script

On success it prints the bounding box, maximum height, base diameter, logo height, solid count, and export paths. On failure it prints an explicit `ERROR:` message.

## Browser 3D preview

Start the local preview server:

```bash
cd /home/miku/projects/chatgpt_icon
python tools/serve_preview.py
```

Then open <http://127.0.0.1:18763/>. The default port is intentionally high (`18763`) to avoid common dev-server conflicts. Override it when needed:

```bash
python tools/serve_preview.py --port 18888
```

The server uses the Python standard library, runs `models/chatgpt_icon.py` once at startup, and watches that file for mtime changes. When the model file changes, exports are regenerated automatically. The browser viewer loads `exports/chatgpt_icon_base.stl` and `exports/chatgpt_icon_logo.stl` as two aligned meshes with different colours, supports rotate/zoom controls, polls `/api/status`, reloads both STLs after successful version changes, and shows captured stdout/stderr if generation fails.

## Screenshot / top-view similarity check

Save the target reference image here:

```text
reference/chatgpt_reference.png
```

Then run:

```bash
python models/chatgpt_icon.py
python tools/compare_top_view.py
```

The comparison uses `exports/preview.png` as the generated top-view screenshot and writes:

```text
exports/similarity_report.json
```

It prints a composite score built from two groups:

- **Shape (82 %):** silhouette IoU, area ratio, edge IoU — how closely the generated silhouette matches the reference.
- **Vector quality (18 %):** `straightness_score` (perimeter stability after morphological smoothing) and `curvature_score` (shape IoU after morphological smoothing) — how clean the edges are: straight lines stay straight, round corners stay round, pixel stair-steps are suppressed.

The `similarity_report.json` includes all sub-metrics: `shape_score`, `vector_quality_score`, `straightness_score`, `curvature_score`, and the weighted `score`. The default “basic close” target is `0.72`:

```bash
python tools/compare_top_view.py --target 0.72
```

When `tools/serve_preview.py` is running, it automatically performs this check after each successful rebuild if `reference/chatgpt_reference.png` exists, and the browser panel shows the latest similarity score. If the score is below target, tune `PETAL_PATH_POINTS`, `PETAL_WIDTH`, `PETAL_START_ANGLE`, and the center void parameters, then save the model file; the preview server will rebuild and re-score.

## Change parameters

Edit the parameter area at the top of `models/chatgpt_icon.py`:

- `BASE_MARGIN_PER_SIDE = 1.5` — margin (mm) between the logo edge and the base edge on each side
- `BASE_THICKNESS = 3.0`
- `BASE_EDGE_FILLET = 0.8`
- `LOGO_HEIGHT = 1.2`
- `PETAL_WIDTH`, `PETAL_PATH_POINTS`, `PETAL_START_ANGLE`
- `CENTER_VOID_SHAPE`, `CENTER_VOID_DIAMETER`, `CENTER_VOID_HEX_FLAT_DIAMETER`

When `TRACE_REFERENCE_LOGO = True`, additional parameters control the bitmap-to-vector cleanup pipeline:

- `TRACE_RESOLUTION = 384` — rasterisation grid size; higher = finer detail but slower
- `TRACE_CLOSE_BUFFER = 0.18` — morphological close (dilate + erode) fills gaps and rounds corners
- `TRACE_OPEN_BUFFER = 0.07` — morphological open (erode + dilate) removes thin noise
- `TRACE_MIN_POLYGON_AREA = 0.50` — discard polygon fragments smaller than this (mm²)
- `TRACE_SIMPLIFY = 0.12` — Douglas-Peucker simplification tolerance; lower = rounder curves

The script validates basic printability constraints: minimum line width is at least `0.8 mm`, central gap is at least `0.5 mm`, and the base encloses the logo with at least `BASE_MARGIN_PER_SIDE` clearance on each edge.

## Import into Bambu Studio

Single-colour print:

1. Import `exports/chatgpt_icon_single.stl`.
2. Slice normally.

AMS / multi-colour print:

1. Import `exports/chatgpt_icon_base.stl` and `exports/chatgpt_icon_logo.stl` together.
2. Keep both files in their original coordinates; do not auto-arrange them separately.
3. Assign the base and logo to different filaments/AMS slots.
4. If Bambu Studio asks whether to load as one object with multiple parts, choose that option.

The STEP file is also available if your CAD/slicer workflow prefers assemblies.

## Adjust logo size and position

- Make the logo larger/smaller: scale the coordinates in `PETAL_PATH_POINTS` together.
- Make strokes thicker/thinner: change `PETAL_WIDTH`; keep it `>= 0.8` for printability.
- Rotate the pattern: change `PETAL_START_ANGLE`.
- Change the centre opening: use `CENTER_VOID_SHAPE = "circle"` or `"hexagon"`, then adjust the matching diameter.
- Keep the logo within the base: the base diameter is computed automatically as the logo max dimension + `2 * BASE_MARGIN_PER_SIDE`.  Increase `BASE_MARGIN_PER_SIDE` for more margin around the logo.
